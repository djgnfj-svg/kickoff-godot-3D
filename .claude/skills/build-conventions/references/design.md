## 적용 범위

UI가 있는 스프린트. CLI·백엔드 전용 Feature에서는 "해당 없음"으로 명시.

## 채우기 가이드

Planner가 첫 UI 스프린트 직전에 아래 카테고리별로 3-5개 규칙을 채운다. 기존 디자인 토큰·Figma·브랜드 가이드가 있으면 그것을 반영한다.

### 카테고리 체크리스트
- [ ] **컬러 팔레트**: 허용 토큰, 금지 패턴 (임의 hex 금지 등)
- [ ] **타이포그래피**: 폰트 스택, 크기 스케일, 행간
- [ ] **간격(spacing)**: 8pt/4pt grid 등 기준, 모바일 증폭 규칙
- [ ] **레이아웃**: breakpoint, 컨테이너 max-width, 그리드
- [ ] **컴포넌트 일관성**: 버튼/입력/카드 등 공통 컴포넌트 사용 의무
- [ ] **상호작용**: hover/focus/active 상태, 애니메이션 duration

## Anthropic이 경계한 "AI 슬롭" 패턴 (참고)

- 보라색 그래디언트 + 플로팅 카드 + 복사된 glassmorphism
- 과도한 이모지·아이콘 장식
- 의미 없는 그림자/블러
- "안전한" 대칭 레이아웃만 반복

규칙을 채울 때 이런 패턴을 명시적으로 금지하는 것을 권장.

### (예시) R1. 컬러는 디자인 토큰으로만
- **선언:** CSS에서 색상은 `var(--color-*)` 토큰으로만 사용. 임의 hex/rgb/hsl 금지.
- **검증:** `rg "#[0-9a-fA-F]{3,6}\\b" src/ --type css` 결과가 토큰 정의 파일 외 0건
- **예외:** 설계 단계에서 명시적으로 등록된 예외만 (이 파일의 예외 목록에 추가)
- **위반 시 판정:** FAIL

### (예시) R2. 간격은 8px grid
- **선언:** margin/padding은 4, 8, 12, 16, 24, 32, 48, 64 중 하나. 그 외 값 금지.
- **검증:** CSS `padding|margin:` 값 점검 (수동 또는 스크립트)
- **예외:** 이미 있는 라이브러리 컴포넌트의 내부 값
- **위반 시 판정:** WARN (첫 도입 시), FAIL (3스프린트 후)
