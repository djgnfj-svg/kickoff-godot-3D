# Sellability Audit Protocol

3D 게임이 Steam(또는 타 스토어)에서 **실제로 팔리기 위한 정량 조건**이 현재 계획에서 충족되는지 감사하는 프로토콜. 주관적 "재밌다"가 아니라 **숫자·경쟁작 사례·Pre-mortem**으로 증명한다.

## 5가지 핵심 감사 축

Sellability Auditor의 `sellability_memo.md`는 아래 5표를 **모두** 채워야 한다.

### 표 1. 가격대 범주 결정

| 범주 | 가격 | 전형적 플레이타임 | 전형적 리뷰 기대 |
|---|---|---|---|
| **초저가 인디** | 1~5$ | 1~3시간 | 매우 긍정적 (85%+) 없으면 실패 |
| **표준 인디** | 5~15$ | 3~15시간 | 매우 긍정적 80%+ 목표 |
| **프리미엄 인디** | 15~30$ | 15~40시간 | 매우 긍정적 80%+ + 스트리머 커버 |
| **AAA-style** | 30~60$ | 40시간+ / 리플레이성 | 매우 긍정적 + 대량 마케팅 |

**작성 규칙:**
- 범주 1개 선택 + 선택 근거 2줄 (경쟁작 가격 평균 + CMD 플레이타임 추정 반영)
- 범주 변경은 컨셉 재검토 급 결정 — 함부로 올리거나 내리지 않는다

### 표 2. Wishlist 목표·손익분기

| 항목 | 수치 | 근거 |
|---|---|---|
| **출시 전 Wishlist 목표** | (예: 10,000) | 경쟁작 출시 직전 Wishlist 중앙값 참조 |
| **Day-1 전환 가정** | (예: Wishlist의 15~20%) | Steamworks 공개 벤치마크 |
| **Day-1 매출 추정** | 가격 × 전환 수 × 0.7 (Steam 수수료 제외) | |
| **손익분기 판매량** | 개발비 ÷ (가격 × 0.7) | 개발비가 0이면 생략 가능 |
| **손익분기 도달 가정 시점** | (출시 N개월) | Wishlist·리뷰 스코어·세일 가정 |

**작성 규칙:**
- 모든 수치는 **가정 1줄 병기**. "가정: Wishlist 전환율 17%, 이유: 장르 평균"
- 손익분기 도달 불가 판정 시 → Founder에게 "가격 재검토 또는 scope 축소" 신호

### 표 3. Day-1 Refund 리스크

Steam은 **플레이 2시간 또는 구매 14일 내** 환불 가능. 첫 2시간에 이탈할 지점을 사전에 3개 식별.

| # | 리스크 지점 | 추정 이탈률 증가 | 대응 제안 |
|---|---|---|---|
| 1 | (예: 튜토리얼 30분 초과) | +5~10% | 튜토리얼 압축 / 옵션 |
| 2 | (예: 초반 난이도 벽) | +5% | 초반 난이도 튜닝 |
| 3 | (예: 컷신 5분 이상) | +3% | 스킵 가능 설정 |

**대표 리스크 후보 (체크하며 훑기):**
- [ ] 튜토리얼·인트로가 코어 버브 첫 수행까지 3분을 넘김 (TTFV 위반)
- [ ] 성능 이슈 (프레임 드롭·크래시)
- [ ] 첫 30분 반복성 부족 (코어 버브가 충분히 반복되지 않음)
- [ ] 초반 UI/UX 혼란 (튜토리얼 없이 키바인딩만 덩그러니)
- [ ] 세이브 손실·진행 롤백 위험
- [ ] 초반 보상 빈도 낮음 (첫 도파민 지점 지연)

### 표 4. Review Score 목표·부정 리뷰 방어

| 항목 | 내용 |
|---|---|
| **목표 스코어 범주** | 긍정적(70%+) / 매우 긍정적(80%+) / 압도적(90%+) 중 1개 |
| **범주 선택 근거** | 경쟁작 범주 + 한 줄 카피 신뢰도 |
| **절대 피해야 할 부정 리뷰 패턴 1** | (예: "10시간 하고 나니 할 게 없다") |
| **절대 피해야 할 부정 리뷰 패턴 2** | (예: "조작이 둔하다") |
| **절대 피해야 할 부정 리뷰 패턴 3** | (예: "가격이 플레이타임 대비 비싸다") |

**작성 규칙:**
- 3개 패턴은 **GMR이 경쟁작 부정 리뷰에서 실제 뽑은 텍스트** 기반 (상상 X)
- 각 패턴마다 "이 패턴이 나오지 않으려면 어떤 설계 결정을 해야 하는가" 1줄

### 표 5. 경쟁작 5개 정량 비교

| 게임 | 가격 | 플레이타임 | 리뷰 스코어 | 동시접속 피크 | DLC 유무 | 출시 후 6개월 추정 매출 |
|---|---|---|---|---|---|---|
| 직접경쟁 A | | | | | | |
| 직접경쟁 B | | | | | | |
| 간접경쟁 C | | | | | | |
| 대체재 D | | | | | | |
| 대체재 E | | | | | | |

**작성 규칙:**
- 출처 URL + 접근일 반드시 병기 (GMR의 research_memo.md §N 참조 링크)
- 데이터 누락 칸은 "N/A — 출처 없음"으로 명시, 추측 금지

## Pre-mortem: 실패 시나리오 3개

**프롬프트:** "출시 후 6개월. 이 게임은 실패했다. 커뮤니티에서 가장 자주 등장한 부정 리뷰 3가지를 지금 적어라."

| # | 부정 리뷰 (가정) | 발생 메커니즘 | 방어 팀원 | 방어 전략 |
|---|---|---|---|---|
| 1 | (예: "코어 버브가 질린다") | 깊이 부족 / 반복 감추는 레이어 부족 | CMD | 깊이 3점 (TTFV/Mastery/Ceiling) 재검토 |
| 2 | (예: "트레일러보다 실제 게임이 떨어진다") | 5초층 훅이 실제 게임에 없음 | HMS | 비주얼 프루프가 in-game에 실제 구현 확인 |
| 3 | (예: "이 가격에는 안 산다") | 플레이타임 대비 가격 불일치 | SA 본인 | 가격 범주 재검토 또는 scope 축소 |

**작성 규칙:**
- 3개는 **서로 다른 축** (기믹·비주얼·가격·UX 중에서 중복 금지)
- 각 방어 전략은 **구체적 설계 행동**. "더 재밌게 만든다" 같은 추상어 금지

## 숨겨진 가정 3개

모든 판매 추정에는 가정이 깔려있다. Phase A 매 사이클마다 **최소 3개** 제출.

**대표 가정 후보:**
- "이 장르가 콜드 시장이 아니라는 가정" — 지난 12개월 출시작 매출 상위 10개가 이 장르에 있는가?
- "MVP 플랫폼(Steam)이 최적이라는 가정" — 장르 특성상 콘솔 먼저가 낫지 않은가?
- "한국인 개발자가 영문/중문 리뷰 Q&A 대응 가능하다는 가정"
- "출시 시점에 경쟁작이 선발매되지 않는다는 가정"
- "엔진(Godot) 선택이 릴리스 빌드 안정성을 보장한다는 가정"
- "에셋 조달이 릴리스 타임라인 안에 끝난다는 가정"
- "한 명의 개발자가 번아웃 없이 완주 가능하다는 가정"

**작성 규칙:**
- 각 가정마다 "가정이 틀렸을 때 최악 시나리오" 1줄
- 틀릴 가능성이 30% 이상이라 판단되면 → how.md Risks 섹션으로 승격 요청

## CMD·HMS와의 경계면 (반드시 충돌)

### vs CMD
**심문 템플릿:**
```
CMD: "이 깊이로 50시간 콘텐츠 가능"
SA: "50시간 콘텐츠의 시장 중앙 가격은 N$ — 우리 범주(M$) 초과"
CMD: "시간당 만족도 근거: {유사 게임 X가 40시간 $30으로 성공}"
SA: "유사 게임 X의 출시 전 Wishlist는 K — 우리가 그 수를 맞출 경로가 있는가?"
```

### vs HMS
**심문 템플릿:**
```
HMS: "이 훅이면 Wishlist 5천 가능"
SA: "경쟁작 5개 중 유사 훅의 Wishlist 중앙값은 2천 — 2.5배 초과 근거는?"
HMS: "{차별 축 근거 또는 채널 가정}"
SA: "그 가정이 틀렸을 때 손익분기 도달 가능한가?"
```

## 체크리스트 (사이클 종료 전 자가 점검)

- [ ] 가격대 범주 1개 + 근거 2줄 완료
- [ ] Wishlist 목표·손익분기 수치 + 가정 병기 완료
- [ ] Day-1 Refund 리스크 지점 3개 + 대응 제안 완료
- [ ] Review Score 목표 범주 + 피해야 할 부정 리뷰 3개 (실제 텍스트 기반) 완료
- [ ] 경쟁작 5개 정량 표 + 출처 URL 병기 완료
- [ ] Pre-mortem 3개 부정 리뷰 + 방어 전략 완료
- [ ] 숨겨진 가정 최소 3개 + 최악 시나리오 1줄씩 완료
