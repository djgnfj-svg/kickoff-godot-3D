# 3D 차원 특화 컨벤션 (Godot 4)

**적용 조건:** `docs/kickoff/_meta.md`의 `project_type`이 `3D 게임 (Godot 4)`인 경우.

이 파일은 `design.md` / `performance.md`의 3D 특화 항목을 모은다. `coding.md` / `accessibility.md`는 차원과 무관하게 공용.

## 디자인 (design)

- **D1. 카메라 거리:** 3인칭이면 타깃과의 고정 범위(예: 3~6m). 1인칭이면 머리 위치 고정. 검증: 카메라 스크립트의 distance/offset 상수.
- **D2. 머티리얼 색 팔레트:** 프로젝트 팔레트 파일(`resources/palette.tres`) 외부 색 사용 금지 (placeholder 예외). 검증: grep으로 `StandardMaterial3D` + `albedo_color = Color(...)` 직접 지정 검출.
- **D3. UI 레이아웃 그리드:** HUD 요소의 화면 safe area (상/하 5%, 좌/우 3%) 준수. 검증: UI 씬의 anchor 값.
- **D4. 폰트/테마:** UI 전체에 지정된 `theme.tres` 사용. 검증: `.tscn`에서 `custom_fonts/` 직접 지정 검출.
- **D5. 피드백 3감각 원칙:** 코어 버브 발동 시 시각/청각/수치 중 최소 2개 동시 반응. 검증: 씬 실행 후 로그에서 `emit_signal("verb_triggered")`마다 대응 이펙트.

## 성능 (performance)

- **P1. 렌더러 선택:** Project Settings의 `rendering/renderer/rendering_method` — Forward+(고품질·PC) / Mobile(모바일·VR) / Compatibility(저사양). 프로젝트 시작 시 `product-spec.md 7항`에 명시. 검증: `project.godot`의 해당 라인이 product-spec과 일치.
- **P2. 타깃 FPS:** PC 60fps (하한), Steam Deck 30fps, VR 90fps (해당 시). 검증: 벤치 씬 평균 FPS.
- **P3. Draw call 한도:** 벤치 씬 프레임당 < 2000 (PC), < 1000 (모바일). 검증: `Performance.get_monitor(...)`.
- **P4. LOD 정책:** 먼 오브젝트(20m+)에 LOD 또는 frustum/occlusion culling. 검증: `VisibilityRange*` 속성 또는 `OcclusionCulling` 활성 여부.
- **P5. HDR/Glow:** WorldEnvironment의 HDR/Glow 사용 시 모바일/Compatibility에서는 비활성 토글. 검증: 환경 리소스 + 플랫폼 토글 코드.
- **P6. 셰이더 복잡도:** 커스텀 셰이더는 `shaders/` 폴더에 한정 + 모바일 타깃이면 vertex/fragment instruction 수 검토. 검증: 폴더 위치 + shader compile log.
- **P7. 메모리 한도:** 벤치 5분 실행 후 증가율 < 10%/분. 검증: `Performance.MEMORY_STATIC`.
- **P8. 씬 로드 시간:** 레벨 씬 로드 < 3초 (PC). 검증: 로드 시작 ~ `_ready()` 완료 스탬프 diff.
