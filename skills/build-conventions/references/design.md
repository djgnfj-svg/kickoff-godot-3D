# Design Conventions (Godot 4 게임)

**상태:** 정적 레퍼런스 (플러그인 유지보수자만 수정)
**적용 범위:** HUD·UI·시각 피드백·카메라가 있는 스프린트.

본 파일의 기본 규칙은 `SKILL.md`의 "design.md (게임 버전)" 섹션 R1~R5(카메라 거리/UI safe area/팔레트 리소스/폰트 일관성/3감각 피드백)를 따른다. 본 파일은 프로젝트별 추가 규칙을 채우는 슬롯이다.

## 채우기 가이드

Planner가 첫 UI 스프린트 직전에 아래 카테고리별로 추가 규칙(0~3개)을 채운다. 기존 디자인 토큰·아트 디렉션이 있으면 그것을 반영한다.

### 카테고리 체크리스트
- [ ] **컬러 팔레트:** `palette.tres` 외 색 사용 정책, placeholder 예외
- [ ] **타이포그래피:** `theme.tres` 의무, 폰트 크기 스케일
- [ ] **HUD 레이아웃:** safe area, anchor 규칙, 모바일/데스크톱 분기
- [ ] **카메라 정책:** 거리·FOV·shake 강도 한도
- [ ] **VFX 정책:** 파티클 한도, 셰이더 복잡도 상한
- [ ] **상호작용 피드백:** hover/focus/press 상태 일관성

## 게임에서 경계할 "AI 슬롭" 패턴 (참고)
- 보라/네온 그라데이션 + 과한 블룸
- 의미 없는 floating particles, lens flare 남용
- HUD 요소 4모서리 가득 채우기 (시야 가림)
- "안전한" 대칭 카메라만 반복

규칙을 채울 때 이런 패턴을 명시적으로 금지하는 것을 권장.

### (예시) RP1. 카메라 shake 한도
- **선언:** 카메라 흔들림은 작은(0.05) / 중간(0.15) / 큰(0.3) 3단 프리셋만. 임의 값 금지.
- **검증:** `grep "shake" scripts/` 결과의 강도 인자가 프리셋 상수인지 확인.
- **위반 시 판정:** WARN

### (예시) RP2. HUD 요소 최대 6개
- **선언:** 한 화면에 동시에 보이는 HUD 요소는 6개 이하. (체력·미니맵·인벤·조준점·메시지·디버그)
- **검증:** HUD 씬의 활성 자식 개수 수동 카운트.
- **위반 시 판정:** WARN
