# 2D 차원 특화 컨벤션 (Godot 4)

**적용 조건:** `docs/kickoff/_meta.md`의 `project_type`이 `2D 게임 (Godot 4)`인 경우.

이 파일은 `design.md` / `performance.md`의 3D 특화 항목(카메라 거리·LOD·셰이더·HDR 등)을 2D 맥락으로 대체한다. `coding.md` / `accessibility.md`는 차원과 무관하게 공용.

## 디자인 (design)

- **D1. 픽셀 퍼펙트:** 도트/픽셀 아트라면 카메라 픽셀 스냅(`Camera2D` `process_callback = PROCESS_PHYSICS` + 정수 위치) 보장. 텍스처 import 시 `Filter = Nearest` 기본값. 검증: `.import` 파일의 `filter=false` 확인.
- **D2. 캔버스 좌표 일관성:** UI는 `CanvasLayer` 위, 월드는 `Node2D` 트리. 두 그룹이 섞이면 위반. 검증: `.tscn`에서 UI 컨트롤이 월드 노드 자식으로 들어가 있는지 grep.
- **D3. Sprite 정렬:** y-sort가 필요한 씬은 `YSort` 사용 (Godot 4: 부모의 `y_sort_enabled = true`). 검증: 해당 씬에 옵션 활성화 확인.
- **D4. 카메라 줌:** 2D 카메라 줌 범위 프로젝트별 고정값 명시(예: 0.5 ~ 2.0). 임의 줌 금지. 검증: 카메라 스크립트의 zoom 상수.
- **D5. UI safe area:** 화면 상/하 5%, 좌/우 3% 안전 영역. 검증: UI 씬 anchor.

## 성능 (performance)

- **P1. 렌더 모드:** Project Settings의 `rendering/renderer/rendering_method = "gl_compatibility"`(저사양) 또는 `"mobile"`. 2D는 Forward+ 불필요. 검증: `project.godot`의 해당 라인.
- **P2. 타깃 FPS:** PC 60fps (하한), 모바일 60fps. 검증: 벤치 씬 평균 FPS 로그.
- **P3. Draw call 한도:** 벤치 씬에서 < 1000 (PC), < 500 (모바일). 2D 스프라이트 배치(MultiMeshInstance2D 또는 atlas)로 줄인다. 검증: `Performance.get_monitor(Performance.RENDER_TOTAL_DRAW_CALLS_IN_FRAME)`.
- **P4. 텍스처 atlas:** 같은 씬에서 자주 쓰이는 스프라이트는 단일 `AtlasTexture` 또는 `SpriteFrames`로 묶는다. 검증: 동일 디렉토리의 분산 PNG 개수 vs atlas 사용 비율.
- **P5. 파티클:** `GPUParticles2D` 우선, `CPUParticles2D`는 호환성 필요 시. 검증: 새 파티클 노드 타입 확인.
- **P6. 메모리 한도:** 벤치 5분 실행 후 증가율 < 10%/분. 검증: `Performance.MEMORY_STATIC`.
