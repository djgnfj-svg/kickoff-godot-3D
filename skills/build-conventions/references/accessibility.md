# Accessibility Conventions (Godot 4 게임)

**상태:** 정적 레퍼런스 (플러그인 유지보수자만 수정)
**적용 범위:** 입력·UI·HUD·자막이 있는 스프린트.

본 파일의 기본 규칙은 `SKILL.md`의 "accessibility.md (게임 버전)" 섹션 R1~R5(키 리매핑/감도·데드존/자막/색약/FPS 선택)를 따른다. 본 파일은 프로젝트별 추가 규칙을 채우는 슬롯이다.

## 채우기 가이드

게임의 접근성은 웹 WCAG와 다르다. 검증 가능한 항목만 규칙화한다 — "친절한 UI" 같은 주관 규칙 금지.

### 카테고리 체크리스트
- [ ] **입력 다양성:** 키보드+마우스, 게임패드, 터치 중 지원 매트릭스
- [ ] **UI 스케일:** HUD 크기 조절 옵션 (작게/보통/크게 3단계)
- [ ] **시각 보조:** 색약 모드, 고대비 모드, 모션 감소 옵션
- [ ] **청각 보조:** 자막, BGM/SFX/UI 분리 볼륨, 시각적 사운드 큐
- [ ] **난이도 보조:** Aim assist, Auto-target, 일시정지 허용
- [ ] **세이브:** 중간 저장 빈도, 체크포인트 정책

## 검증 도구 (Godot 4)
- 수동: 키보드만으로 메인 루프 1회 진행, 게임패드만으로 1회 진행
- 자동: 설정 화면 씬 존재 + 옵션 토글 시 PlayerPrefs 저장 확인 smoke 테스트
- 색약 시뮬레이터: Coblis 등 외부 도구로 캡처 검증 (수동, 기록 의무)

### (예시) RP1. 일시정지 허용
- **선언:** 모든 게임플레이 씬에서 ESC 또는 Start 버튼으로 일시정지 가능. 멀티 전용 모드 제외.
- **검증:** 각 게임플레이 씬의 InputMap에 `pause_toggle` 액션 + 해당 액션이 `get_tree().paused = true` 호출.
- **위반 시 판정:** FAIL

### (예시) RP2. 자막 기본 ON
- **선언:** 첫 실행 시 자막은 기본 ON. 사용자가 끌 수 있어야 함.
- **검증:** `accessibility_options.tres` 초기값 + 설정 씬에 토글 UI.
- **위반 시 판정:** WARN
