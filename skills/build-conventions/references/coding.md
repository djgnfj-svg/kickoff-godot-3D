## 적용 범위

모든 스프린트 (코드가 있는 모든 기능 단위).

## 채우기 가이드

Planner가 첫 스프린트 직전에 아래 카테고리별로 3-5개 규칙을 채운다. 스택과 사용자 프로젝트의 기존 설정을 반영한다.

### 카테고리 체크리스트
- [ ] **폴더 구조**: 기능 단위 경계, 모듈 위치, 공유 코드 위치
- [ ] **명명 규칙**: 파일명·변수·함수·타입·상수
- [ ] **언어/프레임워크 규칙**: TS/Python strict 모드, 런타임 버전, 사용 금지 API
- [ ] **의존성 규칙**: 모듈 간 참조 방향, 외부 패키지 추가 조건
- [ ] **에러 처리**: 예외 전파 정책, 로깅 레벨
- [ ] **테스트**: 최소 테스트 커버리지, 테스트 파일 위치, 테스트 네이밍

## 규칙

<!-- 규칙을 아래 포맷으로 채운다. 예시는 삭제하지 말고 참고용으로 두되, 실제 규칙은 별도로 추가할 것. -->

### (예시) R1. 기능 경계를 넘는 상대 경로 import 금지
- **선언:** `src/features/{A}/...`에서 `src/features/{B}/...`로 직접 import하지 않는다. 공유가 필요하면 `src/shared/`로 승격 후 참조.
- **검증:** `rg "from ['\"]\\.\\./features/" src/features/` 결과 0건
- **예외:** 없음
- **위반 시 판정:** FAIL

### (예시) R2. TypeScript strict 모드
- **선언:** `tsconfig.json`의 `strict: true` 유지. `any` 사용 시 `// @ts-expect-error` + 1줄 사유 주석 필수.
- **검증:** `tsc --noEmit` 오류 0건, 그리고 `rg "\\bany\\b" src/ --type ts` 결과 검토
- **예외:** 외부 라이브러리 타입 없는 경우 `unknown`으로 받고 타입 가드로 좁힘
- **위반 시 판정:** FAIL
